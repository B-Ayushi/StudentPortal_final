Every OCI Service Explained

Each component demonstrates a real cloud concept

👤

User Auth (Module 1–2)

Register and login with hashed passwords. JWT tokens secure every API call. In OCI: use IAM or Identity Domains for enterprise auth.

📝

Project Submission (Module 2–3)

Structured metadata (title, description, tech stack) stored in Oracle Autonomous Database — the managed, self-tuning database on OCI.

📤

File Upload (Module 4)

Files (PDF, ZIP, images) stored in OCI Object Storage. Highly durable, regionally replicated, S3-compatible. Only metadata stays in DB.

🔌

REST API (Module 5)

All actions exposed as HTTP endpoints. In OCI: front the backend with API Gateway for authentication, rate limiting, and CORS policies.

⚖️

Load Balancing (Scaling)

OCI Load Balancer distributes requests across multiple backend VMs. Add more Compute instances to handle thousands of simultaneous submissions.

📊

Monitoring & Logging

OCI Monitoring tracks CPU/memory/latency. Logging centralizes app logs. Alarms notify you when thresholds are crossed.
